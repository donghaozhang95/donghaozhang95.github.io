Leverage Lexical Knowledge for Chinese Named Entity Recognition via Collaborative Graph Network

Posted on 2020-07-20 | Post modified: 2020-07-20 | In 论文 , 期刊或会议 , EMNLP , EMNLP2019 , 领域 , NER , 中文 NER , 方法 , 引入词典 | | Visitors:
Words count in article: 694 | Reading time ≈ 3
为了处理中文命名实体识别中出现的词汇边界信息缺乏的情况，提出了 Collaborative Graph Network，CGN 设计了 3 种图来表示句子中的词汇，而后使用 GAT 来融合图中的信息。该模型可以处理 self-matched lexical words 和 nearest contextual lexical words 的信息。
NER

2. 研究领域与目标

2.1 研究领域

  • 中文 NER

2.2 研究目标

  • 集成词汇知识到句子中所面临的两个问题：
    • self-matched lexical words 的边界信息和语义信息。字符的 self-matched lexical word 是指包含这个字符的词汇。
    • nearest contextual lexical words 的边界信息和语义信息。字符的 nearest contextual lexical words 是指离字符最近的 past 或 future 子序列所形成的词汇。

3. 问题定义

  • 一个字符序列为 \(s=\{ c_1, \dots, c_n \}\)
  • 其中包含的词汇有 \(\{ w_1, \dots, w_m \}\)

4. 方法

4.1 图的构建

4.1.1 Word-Character Containing Graphs (C-graph)

  • 当一个词汇 \(i\) 包含了一个字符 \(j\)，则 \(i\) 与 \(j\) 之间存在一个无向边，在其邻接矩阵 \(A^C\) 中的对应位置处为 1

C-graph

4.1.2 Word-Character Transition Graph (T-graph)

  • 当一个词汇 \(i\) 或一个字符 \(m\) 是一个字符 \(j\) 最近的 preceding 或 following 子序列，则 \(i\) 与 \(j\) 或 \(m\) 与 \(j\) 之间存在一个无向边，在其邻接矩阵 \(A^T\) 中的对应位置处为 1
  • 字符只会连接与之相邻的词组，因为其受到字符之间最小距离为1的限制

T-graph

4.1.2 Word-Character Lattice Graph (L-graph)

  • 受到 lattice lstm 的启发，根据原始 lattice lstm 中字符与词汇之间的连接形式，构建一个图，其邻接矩阵为 \(A^L\)
  • 当一个字符 \(i\) 是字符 \(j\) 最近的 preceding 或 following 的字符，则 \(i\) 与 \(j\) 之间存在一个无向边（意味着两字符相邻，则存在无向边）
  • 另外如果一个字符 \(j\) 是一个词汇 \(i\) 的首字符或尾字符，则 \(i\) 与 \(j\) 之间存在一个无向边

L-graph

4.2 模型的设计

4.2.1 Encoding

  • 针对字符： character embedding + Bi-LSTM，得到 \(H = [h_1, \dots, h_n]\)
  • 针对词汇： word embedding，得到每个词汇 \(w_i\) 对应的 \(v_i\)
  • 节点表示：

\[N_f = [h_1, \dots, h_n, v_1, \dots, v_m]\]

4.2.1 图算法

  • 在 C-graph、T-graph 和 L-graph 上分别使用一个独立的 GAT，更新每个节点的表达，最终获得 \(G_1, G_2, G_3\)

4.2.2 融合层

  • 去除 \(G_1, G_2, G_3\) 中的词汇节点，即保留前 \(n\) 个节点，得到 \(Q_1, Q_2, Q_3\)
  • 然后使用下式融合

\[R=W_1 H + W_2 Q_1 + W_3 Q_2 + W_4 Q_3\]

4.2.3 解码层

  • 使用 CRF 进行解码
