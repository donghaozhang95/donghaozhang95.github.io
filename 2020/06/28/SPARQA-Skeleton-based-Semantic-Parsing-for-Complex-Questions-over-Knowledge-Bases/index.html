SPARQA: Skeleton-based Semantic Parsing for Complex Questions over Knowledge Bases

Posted on 2020-06-28 | Post modified: 2020-06-28 | In 论文 , 期刊或会议 , 领域 , 知识图谱 , 知识图谱问答 , 方法 , AAAI , 语义解析 , AAAI2020 | | Visitors:
Words count in article: 1.9k | Reading time ≈ 7
现有的很多语义解析方法需要依赖于依存树，然而依存树在复杂问句中难以精确产生，存在误差传播现象。因此本文提出了一种 skeleton grammar，一种粗粒度形式的语法，来表示复杂问句更高阶的依赖信息。同时提出了基于 BERT 的 skeleton grammar 解析算法，以及一种用于评估问句与正式查询图之间相似度的多策略方法（结合句子级别的语义与词级别的语义）。

1. 文章信息

  • Sun, Yawei, et al. "SPARQA: Skeleton-Based Semantic Parsing for Complex Questions over Knowledge Bases." AAAI. 2020.
  • AAAI2020
  • GraphQuestions 数据集与 ComplexWebQuestions version 1.1 数据集
  • github

2. 研究领域与目标

2.1 研究领域

  • 知识图谱问答
  • 语义解析

2.2 研究目标

  • 现有的很多语义解析方法需要依赖于依存树，然而依存树在复杂问句中难以精确产生，存在误差传播现象
  • 因此本文提出了一种 skeleton grammar，一种粗粒度形式的语法，来表示复杂问句更高阶的依赖信息

3. 总结

3.1 优点

  • 提出了一种粗粒度的句子结构依存信息表示方法

3.2 不足

  • Skeleton Parsing 过程需要提供人工标注的 Skeleton 结构信息，同时需要使用 4 个 BERT 模型串联起来，效率较低，且存在误差传播的现象
  • sentence-level scorer 和 word-level scorer 要求训练过程需要提供真实的 formal query
  • 论文中提到的
    • 节点识别和链接的误差较高
    • 结构异质性（skeleton 和 ungrounded query 的结构与实际的查询结构不符）的处理
      • 探索一些图算法，如图编辑距离以及图神经网络
    • 无法处理带有聚合函数的问句

4. 问题定义

  • 对于语义解析过程，文中基于 Hu et al. [1] 所提出的 Node-First Framework（NFF）
    • 问句经常转换成一个知识图谱无关的图结构的无根据查询（KB-independent graph-structured ungrounded query），如图 1(c) 所示
    • 然后进一步根据知识图谱转换为一个正式的查询（formal query）（正式的查询可能会具有不同的结构），如图 1(d) 所示

5. 方法

5.1 Skeleton Grammar

  • 可以看做是 dependency grammar 的子集，描述 text span 之间的关系，代表了复杂问句更高阶的依赖结构信息
  • 文中希望实现的是，通过迭代地去除句子的叶子节点，剩余的 text span 总是组成一个最大的合法句子，直到获得了一个不可分的简单的句子，如图 1(b)
  • Skeleton
    • 一个句子的 Skeleton 是一个有向树
    • 节点代表了句子中的 text span
    • 节点之间通过一个边连接，边代表了 text span 之间的依附关系（attachment relation）。同时一个节点依附于另一个节点的 text span 中的某一个词（headword）
  • Text Span
    • 代表了一个短语级别的语义单元（phrase-level）
    • 设置了 4 种类型的 text span：子句（S）、名词短语（NP）、动词短语（VP）、介词短语（PP）
    • 注意在 Skeleton Parsing 过程中，并不需要预测短语属于哪种类型
  • Attachment Relation
    • 定义了在 dependency grammar 常见的 7 种类型的关系：adjectival clause (acl), its sub-type relative clause modifier (acl:relcl), nominal modifier (nmod), its sub-type possessive alternation (nmod:poss), coordination (conj), open clausal complement (xcomp), and adverbial clause modifier (advcl).

5.2 Skeleton Parsing

skeleton parsing

  • 算法过程如上图所示

    • 初始时，\(T\) 由一个节点组成，该节点代表了问句 \(Q\)

    • 然后迭代地分割 \(Q\) 以及增加新的边来扩展 \(T\)，其中涉及到几个关键的函数，其中每个函数都代表了一个 BERT 模型

      • \(\text{Spilt()}\) 函数判断 \(Q\) 是否可分。文中将其建模为 single sentence classification 任务
      • \(\text{TextSpanPrediction()}\) 函数预测从 \(Q\) 中分割的 text span。文中将其建模为 question answering 任务，去除了原始 BERT 模型针对 QA 任务的 question 输入，将 \(Q\) 作为 BERT 模型的 paragraph 输入，然后预测一个在 \(Q\) 中的 text span \(s\)
      • \(\text{HeadwordIdentification()}\) 函数识别在去除 \(s\) 的 \(Q\) 中 \(s\) 所附属的 headword。文中将其建模为一个 QA 任务。将去除 \(s\) 的 \(Q\) 作为 paragraph，\(s\) 作为 question，然后预测一个 span（文中限制为一个词，word）作为 headword \(h\)
      • \(\text{AttachmentRelationClassification()}\) 函数确定 \(h\) 到 \(s\) 之间的关系。文中将其建模为一个 sentence pair classification 任务。将 \(s\) 和去除 \(s\) 的 \(Q\) 作为输入，输出一个预定义的关系类别 \(r\)

5.3 Multi-Strategy Scoring

  • 用来对候选的 grounded queries 进行排序
  • 文中融合了句子级别的信息和词级别的信息
    • sentence-level scorer
      • 探索已知的问句模式到 formal query 模式之间的映射关系（从训练集中获取并保存下来用于测试样本的匹配）
      • 问句模式：将问句中的 entity mentions 替换为虚拟的词（dummy tokens）
      • formal query 模式：将 formal query 中，问句中虚拟词对应的实体，替换为占位符
      • 对于测试样本
        • 找到训练集中与它最相似的，具有相同数目的 dummy tokens 的问句
          • 其中相似的问句模式的评估使用 BERT 模型，同时将其建模为一个 sentence pair classification 任务
          • 在训练时，使用训练集中具有相同问句模式和 formal query 模式的问句作为正样本，使用其他随机问句对作为负样本
        • 然后将该训练集中的问句的 formal query 模式中的占位符，替换为测试样本对应的实体，来生成一个新的 formal query
        • 如果依靠该 formal query 可以检索到实体，则 score 为 1，否则 score 为0
    • word-level scorer
      • 基于 bag of words
      • 将问句和 formal query 表示为词袋（去除了具体的实体以及停词，剩下的词主要表示谓语）（\(Q\) 与 \(P\)，其中 \(Q\) 中 \(n\) 个词，\(P\) 中 \(m\) 个词）
      • 然后用 GloVe 表示两个词袋中的词，同时计算 \(Q\) 与 \(P\) 两个词袋中每个字之间的相似度 \(M=[M_{i,j}]_{n*m}\)
      • 然后选取 \(M\) 中每一行的最大值所对应的位置的所对应的相似度组合作为 \(\mathbf{y}^Q \in \mathbb{R}^n\)，将每一列的最大值所对应的位置的所对应的相似度组合作为 \(\mathbf{y}^P \in \mathbb{R}^m\)
      • 将 \(\mathbf{y}^Q\) 与 \(\mathbf{y}^P\) 送入线性层（此处需要限制 \(n\) 和 \(m\) 以固定线性层的维度）中，输出一个 score

5.4 实验设置

  • 对于节点的识别（包括 entity mention，class，WH words，literals），文中把它当做序列标注的问题。将 Standford NER、SUTime、以及一个基于 BERT 的分类器结合起来，优先选择长的 mention
  • 对于 ungrounded query 向 formal queries 的转换。文中将每一个节点链接为一个知识图谱中的实体、类别或字面值，然后枚举所有可能的相邻节点（adjacent node）之间的谓语
  • 对于链接过程，使用一个编辑自 ClueWeb 的词典以及一个基于知识图谱的资源库。对于 Freebase，使用其 topic names 和 aliases

5.5 误差分析

  • Node Recognition and Linking 占据了 37% 的错误，主要是长的 mention 难以检测
  • Skeleton Parsing 占据了 5%的错误，主要来自 headword 检测
  • Ungrounded Query 占据了 10% 的错误，由于文中所使用的 Hu et al. [1] 中从dependency生成 ungrounded query 的方法所造成的错误
  • Structural Heterogeneity 占据了 22% 的错误，skeleton 和 ungrounded query 的结构与实际的查询结构不符
  • Scoring 占据了 15% 的错误，当训练集中没有测试样本的查询模式时，sentence-level scorer 会出错；当问句模式中词比较少时，word-level scorer 容易出错
